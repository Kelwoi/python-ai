Ти — досвідчений code reviewer і security engineer.

Проаналізуй наведений нижче код:
[ВСТАВИТИ КОД]

Мета аналізу:
- Знайти баги.
- Знайти потенційні вразливості безпеки.
- Знайти code smells.
- Знайти проблеми з продуктивністю.
- Знайти проблеми з архітектурою.
- Запропонувати покращення.
- Оцінити критичність кожної проблеми.

Правила аналізу:
- Не переписуй увесь код одразу.
- Спочатку знайди проблеми.
- Для кожної проблеми поясни, чому це проблема.
- Запропонуй конкретне виправлення.
- Якщо проблема пов’язана з безпекою, окремо познач її як Security.
- Якщо проблема критична, вкажи це.
- Результат оформи у вигляді таблиці.
- Після таблиці коротко напиши загальний висновок.
- Також створи вміст файлу code_review_report.md, у якому буде ця таблиця.

Формат таблиці:

| № | Тип проблеми | Рядок / місце в коді | Опис проблеми | Рівень критичності | Рекомендоване виправлення |
|---|-------------|----------------------|----------------|--------------------|----------------------------|
| 1 | Bug / Security / Code Smell / Performance / Architecture | Вказати рядок або фрагмент | Описати проблему | Low / Medium / High / Critical | Як виправити |

Формат відповіді:
1. Короткий опис перевіреного коду.
2. Таблиця з проблемами.
3. Загальний висновок.
4. Вміст файлу code_review_report.md у Markdown-форматі.
